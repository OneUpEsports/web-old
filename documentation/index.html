Node.js, Yarn and Lerna

Your computer must have installed nodejs and yarn to run this template. You can download Node.js from https://nodejs.org and yarn from https://yarnpkg.com/lang/en/docs/install/ . NPM comes bundled with Node.js








Install Yarn.



Install Npm Packages.

You must have to install packages. You can do this by running yarn bootstrap from the root of your project to install all the necessary dependencies.




Start Development server

Run yarn start from the root of your project to start a dev server. Navigate to http://localhost:8000/. The app will automatically reload if you change any of the source files.




Template Build

Run yarn build to build the project. The build artifacts will be stored in the /public directory.




Run Build Server in local

Run yarn serve this command will serve your static site on the port 9000 (Note: Run this after build)




Build With Path Prefix

If you want to run your website from your cpanel subdirectory. Suppose, your url is `https://hasthemes.com/mitech`. Here your path prefix is `mitech`. Then you have to look at this documentation page. (https://www.gatsbyjs.com/docs/how-to/previews-deploys-hosting/path-prefix/)



Fix Linting and Prettier Error

We have used eslint and prettier to manage and format our coding style. If you have any linting error Run yarn lint:fix to fix eslint error.




Clean App

Run yarn clean will remove your .cache and other unnecessary files

Customize GraphQL Schema





In this template we have customized graphql schema. That means all the data types are defined in our typedefs. if you want to add a new field in json or markdown then you have to define the typedefs first.

To know more about gatsby schema generation read these documentation
https://www.gatsbyjs.com/docs/reference/graphql-data-layer/schema-customization/
https://www.gatsbyjs.com/docs/schema-generation/
https://www.gatsbyjs.com/docs/schema-inference/
